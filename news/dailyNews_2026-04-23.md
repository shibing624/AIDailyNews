### [Karpathy风格代码改进指南：提升Claude代码行为](https://github.com/forrestchang/andrej-karpathy-skills)

来源：Trending repositories on GitHub this week · GitHub

发布时间：2026-04-23 00:12:07

本文基于Andrej Karpathy对LLM编程缺陷问题的分析，提出了四大原则以优化Claude Code行为：1. 在编码前思考：明确假设、处理混乱并呈现权衡；2. 优先简洁：避免过度工程，减少代码冗余；3. 精准修改：减少对非相关代码的非必要变动；4. 目标导向执行：定义明确成功标准，逐步验证。这些原则通过插件形式提供，帮助开发者减少无效改动，优化代码简洁性和功能性，同时鼓励为复杂任务设置验证循环。该指南对提升代码质量及减少错误改动具有重要意义，对于需要快速和高质量代码交付的开发者来说尤为实用。

### [GenericAgent：极简的自我进化LLM自主框架](https://github.com/lsdefine/GenericAgent)

来源：Trending repositories on GitHub this week · GitHub

发布时间：2026-04-23 00:12:07

GenericAgent是一个极简的自主Agent框架，核心代码仅约3000行，通过9个原子工具及约100行的自我进化循环，支持对浏览器、终端、文件系统等的系统级控制。其自我进化机制将代理每次完成的复杂任务固化为技能，随着使用时间增长形成用户专属技能树，支持主流LLM进行协作运行。该框架旨在避免过度工程，支持动态创建工具以完成多样化任务。示例功能包括自动化股票筛选、批量消息发送等高效任务操作。GenericAgent的特点在于极简部署、高功效执行和令牌消耗的经济性，对需要长期累积任务效率的用户尤为合适。
